# Groundhog

Pet-проект для отслеживания рабочего времени (time tracker), разработанный с целью получения практического опыта интеграции **Claude Code** в реальный процесс разработки.
Название отсылает к фильму "День сурка": каждый день повторяется — пока не начнёшь им осознанно управлять.

## Цель проекта

Основная цель — не сам трекер времени, а изучение того, как Claude Code встраивается в повседневный рабочий процесс разработчика:

- Постановка задач через диалог с Claude Code
- Архитектурные решения в паре с AI-ассистентом
- Code review и рефакторинг с помощью Claude Code
- Написание тестов и документации совместно с AI

## Функциональность

- Запуск и остановка таймера для задач
- Привязка записей времени к проектам и тегам
- Просмотр истории с фильтрацией по дате / проекту
- Экспорт отчётов (CSV / JSON)

## Составные части

### Backend API
Сервис на **C#/.NET** — основа системы, обрабатывает всю бизнес-логику и хранение данных.

### Web Frontend
Веб-интерфейс для полноценной работы с трекером: просмотр истории, отчёты, управление проектами и тегами. Стек: React + Vite + TypeScript + Tailwind + shadcn/ui.

### macOS App
Миниатюрное приложение для быстрого доступа: запустить или остановить таймер одной кнопкой, не отвлекаясь от работы. Стек: Rust/Tauri.

## Структура репозитория

```
/
├── backend/    # C#/.NET — API и бизнес-логика
├── frontend/   # React + Vite + TypeScript — веб-интерфейс
├── desktop/    # Rust/Tauri — macOS-приложение
└── docs/       # OpenAPI-схема, архитектурные решения
```

## Разработка

Проект ведётся итеративно: каждая фича обсуждается и реализуется в диалоге с Claude Code. История решений фиксируется в [DECISIONS.md](./DECISIONS.md).
